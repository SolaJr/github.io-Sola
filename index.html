My main phases design system

Phase 1 - People Problem (Analyze Requirements)

 If a problem involves people, this phase is nessary.  My 4 steps are : find the people problem, discover its value, define success and reflection. I adapted this method from a procedured presented by Julie Zhuo. This helps with the ideation stage in phase 2.

  • 1. Find the people problem

    What is the people problem we are trying to solve that gave us the problem statement?

  • 2. Discover its value

    How do we know this is a real problem?

  • 3. Define success

    How will we know if we've solved this problem?

  • 4. Reflection

    How can we use what we learned to expand the potential of this solution?

Phase 2 is the typical product design cycle.  Analyze Requirements, Design Solutions, Implementation and Testing, and Evaluation. This cycle repeats after evalutation. 

  • 1.  Analyze Requirements

    Use the research from phase 1 and modern research in the problem's enviorment to develop requirements. Typically includes relevent psychology research. 

  • 2. Design Solutions

    Develop multiple possible solutions and differen't methods to test each one. Keeping in mind the requirements found in phase 1.

  • 3. Implementation and Testing

    Develop a plan to implement the solution into the current system. Also create testing parameters for usablity testing to see how people interact with it.  Depending on the nature of the problem, it can range from surveying pure strangers or using websites like Usertesting.com .

  • 4. Evlauation

    Evaluate the results with the criteria developed in phase one. Then idenifiy if their are any new findings or places of intrest that need to be reconsidered.
